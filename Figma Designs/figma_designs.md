# Figma Designs – Real Estate Marketplace

## Introduction

This document describes the **UI/UX designs for the Real Estate Marketplace platform** created using Figma.

The designs illustrate how users interact with the system to browse properties, list properties, schedule visits, negotiate prices, and complete purchases. The Figma prototype serves as the visual blueprint for implementing the front-end interface of the application.

# Key Screens in the Design

## 1. Landing Page

Purpose:
The landing page serves as the entry point of the platform and introduces users to the marketplace.

Main components:

* Navigation bar
* Search bar for properties
* Featured property listings
* Login / Sign up buttons
* Platform introduction section
* Footer with useful links

---

## 2. Login Page

Purpose:
Allows registered users to access their accounts.

Components:

* Email input field
* Password input field
* Login button
* Forgot password option
* Link to registration page

---

## 3. Registration Page

Purpose:
Allows new users to create an account on the platform.

Fields included:

* Name
* Email
* Phone number
* Password
* Role selection (Buyer / Seller / Agent / Admin)

---

## 4. Property Listing Page

Purpose:
Displays all available properties on the platform.

Features:

* Property cards with images
* Price and location information
* Filters for property type and price
* Sorting options
* Pagination for browsing multiple properties

---

## 5. Property Details Page

Purpose:
Shows detailed information about a selected property.

Key elements:

* Property image gallery
* Property description
* Price and specifications
* Property location
* Contact seller option
* Shortlist button
* Schedule visit button
* Property document

---

## 6. Seller Dashboard

Purpose:
Allows sellers to manage their property listings.

Features:

* Add new property
* Edit property details
* Upload property images
* Upload property documents
* View listing status

---

## 7. Buyer Dashboard

Purpose:
Allows buyers to track their interactions with properties.

Features:

* View shortlisted properties
* Track scheduled visits
* Manage negotiations
* View purchase history

---

## 8. Visit Scheduling Interface

Purpose:
Allows buyers to schedule visits for properties they are interested in.

Fields:

* Select property
* Choose date
* Choose time slot
* Confirm visit request

---

## 9. Negotiation Interface

Purpose:
Allows buyers and sellers to negotiate property prices before finalizing the purchase.

Features:

* Submit price offers
* Counter offers
* View negotiation status
* Accept or reject offers

---

## 10. Admin Dashboard

Purpose:
Allows administrators to monitor and manage the platform.

Features:

* Verify property documents
* Manage user accounts
* Handle reported users
* Monitor transactions and payments

---

# User Navigation Flow

The general navigation flow in the application is:

Landing Page
→ Property Listings
→ Property Details
→ Shortlist / Schedule Visit
→ Negotiation
→ Purchase and Payment

Users can also access role-specific dashboards depending on whether they are buyers, sellers, agents, or administrators.

---

# Design Principles

The UI design follows the following principles:

* **Simplicity** – Easy to understand interface
* **Consistency** – Similar layout patterns across pages
* **Accessibility** – Clear navigation and readable text
* **Responsiveness** – Suitable for different screen sizes

---

# Design Notes

* The Figma designs serve as the visual blueprint for front-end development.
* The interface aligns with the system architecture and database design defined in the ER diagram.
* These designs help developers understand how different components interact within the system.
